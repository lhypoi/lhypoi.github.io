hexo博客搭建

本地配置

  • node安装

    • node官网：https://nodejs.org/zh-cn/，按照步骤一路装下去就可以了
    • 测试安装情况：cmd输入指令node -v，出现版本号即成功

  • git安装

    • git下载地址：https://git-scm.com/download，安装过程中Git Bash Here和Git GUI Here要记得勾选，用于文件夹中右键打开git界面
    • 测试安装情况：右键选择Git Bash Here，输入git --version，出现版本号即成功

  • hexo cli安装

    • cmd输入指令npm install -g hexo-cli

基本指令

以下指令均在Git Bash Here窗口执行

  • 新开一个博客项目：hexo init 博客项目文件夹名
  • 安装hexo项目编译运行需要的依赖包：npm install
  • 新建一篇博客：hexo new 博客标题,markdown文件的默认路径：\source\_posts\...
  • 编译markdown生成静态文件：hexo g
  • 本地服务器运行预览：hexo s,默认地址：http://localhost:4000/
  • 上传到gitPage：hexo d，环境和参数配置见下方

deloy配置

以下指令均在Git Bash Here窗口执行

  • git本地配置

    • 配置git邮箱：git config --global user.email "(你注册github的邮箱)"
    • 配置git用户名：git config --global user.name "(你github的用户名)"
    • 生成秘钥：ssh-keygen -t rsa -C "(你注册github的邮箱)"，回车三次
    • 复制秘钥，下面用到：clip < ~/.ssh/id_rsa.pub

  • git云端配置

    • 点击左上角的猫，回到首页，点击New repository按钮，进入创建仓库界面，填写Repository name的时候，Repository name的格式必须为{user_name}.github.io，其中{user_name}必须与你的用户名一样，这是GitHub Pages的特殊命名规范，并且需要勾选Initialize this repository with a README
    • 在建好的项目右侧有个settings按钮，点击它，向下拉到GitHub Pages，选择master分支，保存。输入https://你的用户名.github.io/测试能否访问
    • 点击当前页面左边的Deploy keys选项，新建一个SSH key，填写标题Title给这个ssh key起个名字，并将上面复制的秘钥粘贴到Key输入框，之后按下添加秘钥按钮
    • 测试SSH成功与否：ssh -T git@github.com
    • 回车，出现are you sure you want to continue connecting?时输入yes
    • 回车，如果出现hi,(你github的用户名),you are successfully ·······即成功
